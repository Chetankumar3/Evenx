3. AUTH — single spec, implement natively in BOTH main and statesync
--------------------------------------------------------------------------------
No shared auth file. Implement this exact logic twice, once per language.
Shared between them: only the JWT_SECRET env var value and the Redis instance.

Claims:  { sub: <userId>, jti: <uuid>, iat, exp }   exp = iat + 2h, HS256.

main — issue (on /register and /login success):
  pseudo:
    token = jwt.sign({ sub: user.id, jti: uuid4() }, JWT_SECRET, { expiresIn: '2h' })
    respond 200 { token }

main — verify (Express middleware, every route EXCEPT GET /events, 
  GET /event_details/:id and GET /search):
  pseudo:
    token  = req.headers.authorization?.replace('Bearer ', '')
    if (!token) return res.status(401).end()
    claims = jwt.verify(token, JWT_SECRET)              // 401 on throw
    if (await redis.exists(`jwt:denylist:${claims.jti}`)) return res.status(401).end()
    req.userId = claims.sub
    next()

main — logout:
  pseudo:
    ttl = claims.exp - nowInSeconds()
    await redis.setex(`jwt:denylist:${claims.jti}`, ttl, "1")
    respond 200

statesync — verify (first msg auth prototocl, accept all ws, then use first message for auth with 10s timeout):
  pseudo:
    claims, err := jwt.Verify(token, JWT_SECRET)
    if err != nil { reject upgrade with 401 }
    exists, _ := redisClient.Exists(ctx, "jwt:denylist:"+claims.Jti).Result()
    if exists > 0 { reject upgrade with 401 }
    conn.userID = claims.Sub

GET /events, GET /event_details/:id and GET /search are the ONLY three 
unauthenticated routes anywhere in the system. Every other route, every WS 
connection, requires a valid, non-denylisted JWT.
Auth in WS connection happens in First msg. So, accept all ws, then use first message for auth.

4. REDIS SCHEMA — exact keys, do not rename or restructure
--------------------------------------------------------------------------------
Shared by both booking models, created when an event is seeded or lazily on
first access if missing (see Sec.9, rehydration):
  <eventid>:model:<type>   string, 'general' | 'seat_map'
  <eventid>:total          int, = events.total_seats
  <eventid>:avlbl          int, init = total
  <eventid>:book           int, init = 0
  pubsub channel <eventid> broadcasts state deltas, both models use it

seat_map model ONLY:
  <eventid>:status         Bitmap, 2 bits/seat: 00=empty 01=locked 10=booked
  <eventid>:lock:<seat_num>    ephemeral string, value=<userid>, EX=LOCK_TTL
                           (LOCK_TTL = env variable)
  keyspace notif on <eventid>:lock:<seat> expiry

general model ONLY:
  <eventid>:lock:<count>:<userid>   ephemeral string, value="true",
                                     EX=LOCK_TTL (10min)
  keyspace notif on <eventid>:lock:<count>:<userid> expiry
  IMPORTANT: count and userid live IN THE KEY NAME on purpose. A Redis
  keyspace-expiry notification gives you the key name only, never the value
  that just expired — encoding count+owner in the name is how restorer
  recovers both on expiry without a second non-expiring lookup table. Do
  not "simplify" this back to <eventid>:lock:<userid>=N — that breaks expiry
  recovery.

6. STATESYNC — WS protocol & Lua Script (Locking)
--------------------------------------------------------------------------------
Endpoint: GET /stream/:eventid (upgrades to WS after Sec.3 auth check).
On connect:
  seat_map -> send { type: "INIT", bitmap: <string> }
  general  -> send { type: "INIT", avlbl: <int>, book: <int> }
  Then subscribe the connection to Redis pubsub channel <eventid>.

Client -> server messages:
  seat_map: { reqid, action: "LOCK" | "UNLOCK", seat_num: [int, ...] }
  general:  { reqid, action: "LOCK", num_seats: <int> }
  BOTH:     { reqid, action: "DONE" }   <- graceful-close handshake, see below

LUA SCRIPT PSEUDOCODE FOR LOCKING (Run via EVAL):
  seat_map LOCK: 
    For each `seat` in payload:
      if bitmap[seat] == 00 (empty):
        bitmap[seat] = 01 (locked)
        SETEX <eventid>:lock:<seat_num> = <userid>
        avlbl -= 1
      else:
        abort the entire script, return failure (atomic rollback).
  general LOCK:
    if avlbl >= num_seats:
      avlbl -= num_seats
      SETEX <eventid>:lock:<num_seats>:<userid> = "true"
      return success
    else: 
      return failure

Server -> client messages:
  LOCK ack:    { reqid, success: bool, avlbl?: int, failed_seats?: [int,...] }
  UNLOCK ack:  { reqid, success: bool }
  pubsub delta, seat_map: { seat_num: int, new_status: 0|1|2 }
  pubsub delta, general:  { avlbl: int, book: int }

GRACEFUL-CLOSE HANDSHAKE (applies to both models, implement exactly this):
  1. Client sends a successful LOCK.
  2. Client sends { reqid, action: "DONE" }.
  3. Server marks this connection's locks as "released" internally (a bool
     flag on the connection struct), does NOT touch Redis for this.
  4. Server then closes the socket.
  5. On WS close: if the "released" flag is set, do nothing further. If it
     is NOT set (no DONE received), run abandonment cleanup — release every 
     lock this connection is holding, immediately (using manual UNLOCK). Each WS  connection maintains an in-memory set of currently-held seat locks
for seat_map events and the currently-held count-lock for general events.

Successful LOCK adds ownership to this set.
Successful UNLOCK removes ownership from this set.

Disconnect cleanup operates only on the locks still present in this
connection-local ownership set.

Reconciler: one goroutine per active event, runs every 30s. Get the <eventid>:status for seat_map model and <eventid>:avlbl for general model to refresh the state.
general: SCAN <eventid>:lock:*, sum counts parsed from key names,
  recompute avlbl_check = total - book - sum, fix drift, push.

8. RESTORER — pseudocode, the whole service
--------------------------------------------------------------------------------
  subscribe to "__keyevent@0__:expired"
  for each expired key name K:
    parts = K.split(":")
    eventId = parts[0]

    if parts[1] == "lock" and len(parts) == 3:        # seat_map: ev:lock:<seat_num>
      seatNum = parts[2]
      pipeline:
        bitfield SET <eventId>:status seat -> 0
        INCR <eventId>:avlbl
      publish(eventId, { seat_num: seatNum, new_status: 0 })

    elif parts[1] == "lock" and len(parts) == 4:       # general: ev:lock:<N>:<userid>
      n = int(parts[2])
      INCRBY <eventId>:avlbl n
      publish(eventId, { avlbl_delta: n })

  No DB calls anywhere in this service. No JWT/auth code anywhere in this
  service. If you find yourself adding either, stop — that's a sign you've
  drifted from spec.

9. REHYDRATION — when Redis state is missing
--------------------------------------------------------------------------------
Triggered in statesync, on connect, if <eventid>:total is missing:
  total = events.total_seats (from DB)
  if event.model == 'general':
    book = SUM(bookings.num_seats WHERE event_id=eventId AND status='confirmed')
  else:
    book = COUNT(seats_booked WHERE event_id=eventId AND cancelled=false)
  avlbl = total - book
  SET <eventid>:total, <eventid>:avlbl, <eventid>:book accordingly.
  if seat_map: also rebuild the bitmap string — set bit=2 for every seat_num
  in seats_booked for this event (where cancelled=false), bit=0 for every other seat.
